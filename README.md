# Project 2 - Swiggy Mobile App Testing (Android)

##  Application Under Test
Swiggy Mobile Application (Android)

##  Objective
To validate core food ordering functionalities including:
- App launch
- Location access
- Restaurant browsing
- Adding items to cart
- Cart updates
- Checkout process
- Payment handling
- UI/UX behavior

##  Test Artifacts Included
- Test Plan (DOCX/PDF)
- Test Scenarios
- 20 Test Cases (Excel)
- Bug Report (8 bugs)
- Test Execution Report
- Screenshots (Execution Evidence)

##  Testing Types Performed
- Functional Testing
- UI Testing
- Usability Testing
- Negative Testing
- Compatibility Testing

## Key Bugs Found
### Critical Bug: Payment Failure
- App displays "Your payment failed" with no retry or support options
- User unable to complete order
- Major business impact  
(Screenshot: `06_Payment_Failure_P2.png`)

### UI Bug: Misaligned ADD Button
- Long item names cause ADD button to shift downward
- Inconsistent layout and spacing  
(Screenshot: `07_UI_Misalignment_P2.png`)
